# 毕业设计答辩 PPT 大纲

## 1. 封面

题目：基于 C++ 高并发架构与语言大模型的交互系统的开发

展示内容：学生姓名、专业班级、指导教师、学院、日期。

## 2. 研究背景

说明大语言模型应用快速发展，智能问答系统需要同时具备自然语言交互能力和稳定的并发接入能力。传统单一脚本服务开发快，但在连接管理、并发调度和系统分层方面存在不足。

## 3. 需求分析

用户侧需求：

1. 用户登录。
2. 发起智能问答。
3. 查看历史记录。

管理员侧需求：

1. 查看系统概览。
2. 查看系统日志。
3. 维护模型配置。

非功能需求：并发处理、稳定性、可维护性、跨环境运行。

## 4. 系统架构

展示图4-1系统总体结构图。说明系统分为表现层、C++ 接入层、Python AI 服务层和数据层。

答辩重点：

1. C++ 网关负责高并发连接接入和请求转发。
2. Python FastAPI 服务负责业务接口和模型调用。
3. 数据层负责会话、日志和配置持久化。
4. 前端提供用户页面和管理员页面。

## 5. 关键实现

展示 C++ 网关实现：

1. epoll 事件监听。
2. Reactor 请求分发。
3. 线程池任务处理。
4. 按 Content-Length 读取完整 POST 请求体。
5. 上游不可用时返回 502 Bad Gateway。

展示 Python 服务实现：

1. `/api/login` 登录。
2. `/api/chat` 问答。
3. `/api/history` 历史记录。
4. `/api/admin/overview` 管理概览。
5. `Repository` 抽象支持 memory 与 SQLite。

## 6. 跨环境运行

展示三类运行路径：

1. Windows：前端 + Python 服务，用于快速演示主业务链路。
2. WSL：前端 + Python 服务 + C++ epoll 网关，用于验证高并发专项模块。
3. Linux：复用 WSL 编译、启动和压测命令，用于部署和性能实验。

## 7. 测试结果

功能测试：

1. 登录测试通过。

并发测试：
